# 기획 시작

## 사용 예시
```
# Jira 이슈 기반
/기획시작 Jira: SOLID-123

# Jira + 추가 의견
/기획시작 Jira: SOLID-123 / 추가: JWT 인증 사용, 소셜 로그인은 2차로

# Jira 없이 직접 요구사항
/기획시작 로그인 기능 만들어줘. 이메일+비밀번호 방식, JWT 사용

# 여러 Jira 이슈 묶어서
/기획시작 Jira: SOLID-123, SOLID-124 / 추가: 두 기능을 하나의 기획서로 통합
```

---

아래 순서로 진행한다.

## 1. 에이전트 하네스 로드
`agents/planning.md` 파일을 읽고 기획 에이전트 규칙을 완전히 숙지한다.

## 2. Jira 이슈 조회 (이슈 번호가 있는 경우)
입력에 Jira 이슈 번호가 포함된 경우 Jira MCP로 해당 이슈를 조회하여
- 이슈 제목, 설명, 담당자, 우선순위
- 연결된 Epic, 하위 이슈
- 기존 댓글 및 첨부 정보

를 가져온다.

## 3. 입력 정보 통합
아래 입력 정보를 통합하여 기획 초안의 기반으로 사용한다.

```
$ARGUMENTS
```

통합 기준:
- Jira 이슈 내용 → 공식 요구사항 및 배경
- 희준님 추가 의견 → 기술 방향, 우선순위, 제약사항, 제외 항목

## 4. 기획 에이전트 시작
`agents/planning.md` 규칙에 따라 5개 서브 에이전트 토론을 시작한다.

- 토론 전 통합된 입력 정보를 PM 에이전트에게 전달
- PM → Devil's Advocate → UX → 기술 → 보안 순으로 토론 진행
- 합의 완료 후 `docs/기획/{기능명}.md` 저장

## 5. 완료 보고
기획문서 작성 완료 후 오케스트레이터에게 보고한다.

```
## 기획 완료 보고
- 산출물: docs/기획/{기능명}.md
- 주요 결정사항: {요약}
- 미결 사항: {있을 경우}
- GATE 1 승인 요청: "기획 승인" 또는 "설계 시작해" 명령을 입력해주세요
```
